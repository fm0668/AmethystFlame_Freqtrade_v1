# V2.4因子说明（含参数与计算逻辑）

## 1. 总体框架
- 频率：4H
- 横截面对象：全可交易USDT永续
- 输出：每个4H时点做多Top5、做空Top5
- 标签：未来4H收益 `ret_fwd_4h`
- 训练方式：滚动训练窗口（84个4H bar）
- 市场状态：BTC强/弱分组（regime）分别训练参数

## 2. 基础变量
- `ret_4h`：4H收益率
- `ret_24h`：24H收益率（6个4H）
- `ret_fwd_4h`：未来4H收益率（标签）
- `vol_4h_surge`：4H成交额相对过去42根均值放量比
- `taker_buy_ratio`：主动买入成交额占比
- `d_oi`：OI变化率
- `funding_rate`：资金费率
- `global_ls_ratio`：全市场多空比
- `top_ls_ratio`：大户多空比

## 3. 扩展特征
- `breakout_20`：`close / rolling_high_20 - 1`
- `rebound_20`：`close / rolling_low_20 - 1`
- `trend_hit_6`：最近6根4H上涨占比
- `pullback_quality`：相对近6根高点回撤质量
- `upper_wick`：上影线占比
- `lower_wick`：下影线占比
- `d_top_ls_24h`：大户多空比24H变化
- `d_global_ls_24h`：全市场多空比24H变化
- `whale_exit_raw`：`max(ret_4h,0) * max(-d_top_ls_24h,0)`
- `oi_bear_div_raw`：`max(ret_4h,0) * max(-d_oi,0)`
- `activity_oi_div_raw`：`max(vol_4h_surge,0) * max(-d_oi,0)`

## 4. 标准化
- 所有核心特征按每个4H时点做横截面Z-Score：
  - `z_x = (x - mean_cs) / std_cs`
  - 再做裁剪：`clip[-3, 3]`

## 5. 做多因子定义（V2.4）
- `L1_momo`：相对BTC动量 `z_mom_rel`
- `L2_flow`：资金流强度 `0.5*z_vol_4h_surge + 0.5*z_taker_rel`
- `L3_confirm`：趋势确认 `0.5*z_breakout_20 + 0.5*z_trend_hit_6`
- `L4_oi`：持仓确认 `z_d_oi`
- `L5_cost`：拥挤成本 `(-z_funding_rate - z_global_ls_ratio)/2`
- `L6_rebound`：反弹质量 `0.6*z_rebound_20 + 0.4*z_pullback_quality`
- `L7_candle`：K线下影强度 `z_lower_wick`

## 6. 做空因子定义（V2.4）
- `S1_crowd`：拥挤度 `(z_funding_rate + z_global_ls_ratio + z_top_ls_ratio)/3`
- `S2_whale_exit`：大户撤退 `z_whale_exit_raw`
- `S3_price_oi_div`：价升OI降背离 `z_oi_bear_div_raw`
- `S4_activity_oi_div`：活跃升OI降背离 `z_activity_oi_div_raw`
- `S5_upper_wick`：上影线衰竭 `z_upper_wick`

## 7. 动态因子权重
- 做多组件：`[L1..L7]`
- 做空组件：`[S1..S5]`
- 每个训练窗口动态计算每个因子的稳定性分数：
  - 计算每个4H截面的加权Spearman相关
  - 统计 `mean_ic / (std_ic + 0.05)` 并乘以正向占比
  - 负值截断为0
  - 归一化为最终因子权重

## 8. 样本权重（强调前排）
- 对每个4H时点真实Top30样本赋权：
  - 做多：`w_long = exp(-0.35*(rank_up-1))`
  - 做空：`w_short = exp(-0.25*(rank_dn-1))`
  - Top30外给很小基线权重（做多0.005、做空0.01）

## 9. 参数阈值（动态分位）
- 做多阈值（在训练窗口按加权分位提取）：
  - `long_vol_4h_min`：`vol_4h_surge`的20%分位
  - `long_breakout_min`：`breakout_20`的20%分位
  - `long_trend_min`：`trend_hit_6`的25%分位
  - `long_rebound_min`：`rebound_20`的20%分位
- 做空阈值：
  - `short_funding_min`：`funding_rate`的35%分位
  - `short_global_min`：`global_ls_ratio`的35%分位
  - `short_top_min`：`top_ls_ratio`的35%分位

## 10. 置信阈值与候选搜索
- 搜索区间：
  - 做多分位 `long_conf_q ∈ {0.15, 0.20, 0.30}`
  - 做空分位 `short_conf_q ∈ {0.25, 0.35}`
  - 做空连涨最小bar `n ∈ {3, 6}`
- 目标函数（偏做多）：
  - `obj = 0.70*long_ret + 0.30*short_ret - 0.20*long_dd - 0.10*short_dd`

## 11. 选币逻辑
- 先按阈值筛池，再按综合分排序取Top5
- 若不足最小持仓数（3）：
  - 用全池高分标的补位
- 做空额外预筛：
  - 先取当根`ret_4h`涨幅前80
  - 再应用连涨bar阈值

## 12. 输出字段说明
- `score`：综合打分（动态权重后的加权和）
- `real_top30_rank`：真实涨跌幅Top30排名
- `ret_fwd_4h`：未来4H收益
- `max_drawdown_4h`：
  - 做多：`next_low/close - 1`
  - 做空：`1 - next_high/close`

## 13. 注意事项
- 所有计算使用“已闭合4H bar”，避免未来函数
- 资金费率缺失时需先完成API补充，否则因子稳定性会下降
- 若迁移到执行框架，建议保持“筛选层与执行层解耦”
